Monolithic architecture

  1. Monolithic architecture contains all modules in one place
  2. Has some benfits on classical small applications, but has far many drawbacks compared to the benefits
  3. Scalability issues (hard to add new big modules)
  4. Reliability issues (one bug will affect the availability of whole application)
  5. Availability issues (if one service fails, the whole application has to be brought down)
  6. Agility issues (even small component change requires reassambling the application)
  7. Software Stack Autonomy issues (has barrier on implementing different stacks)
  8. Issues with new team members (takes long time for getting to know whole application)

Microservices are solution

Microservice architecture

  1. Solves problems by dividing app into multiple independent services
  2. Services talk with each other through REST APIs
  3. Each service can be developed on different software stack
  4. However, has some drawbacks!
  5. Complex architect design
  6. Hard to deploy
  7. Complex testing
  8. Multiple database updates

What to choose?

  1. Highly depends on the scale of the project
  2. Prefer monolith for simple and small projects
  3. Microservice can be better solution for large and long term projects

Deployment issues?

How to solve the microservice deployment issues?

Docker is the solution!

  1. Enables task isolation by creating docker container for each microservice
  2. Supports multiple coding languages required to run services
  3. Enables database separation

Why not virtual machines!

  1. There is option for deploying microservises application on virtual machines
  2. However, virtual machines are not the best option

Virtual machines vs Docker

  1. Docker is more effective when it comes to resource allocation
  2. Docker is easier to deploy
  3. Docker has smaller size
  4. Docker has much faster startup time
